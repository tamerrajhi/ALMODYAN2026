ROLE: Full-Stack Engineer

GOAL
ربط إعدادات ZATCA (zatca_mode + integration_enabled) مع كل نقاط الاستخدام في الفواتير (POS + ERP) بحيث:

1) sandbox + integration_enabled = false
   → Virtual Mode ONLY:
   - يسمح بتوليد/عرض الـ artifacts محلياً (zatca_status, qr/xml preview إن وجد) بدون أي إرسال للهيئة
   - زر الإرسال يكون Disabled مع Tooltip واضح: "وضع تجريبي (Virtual) — فعّل التكامل للإرسال"

2) sandbox + integration_enabled = true
   → يسمح بالإرسال الحقيقي إلى بيئة Sandbox عبر Edge Functions

3) production + integration_enabled = false
   → يمنع الإرسال (Disabled) + رسالة واضحة

4) production + integration_enabled = true
   → يسمح بالإرسال الحقيقي ONLY إذا كانت متطلبات الشهادة/الـ onboard مكتملة
   - إذا ناقص شيء → Hard-block + رسالة ما ينقصه

IMPLEMENTATION RULES
A) Single Source of Truth
- useZatcaSettings يرجّع دائماً:
  { zatca_mode: 'sandbox'|'production', integration_enabled: boolean, branch_id?, cert_status? ... }
- كل UI components تعتمد على hook فقط (لا تعتمد على local state لنفس القيم)

B) UI Behavior
- ZatcaSubmitButton:
  - يقرأ الإعدادات من useZatcaSettings
  - يحسب canSubmit = (integration_enabled === true) AND (if production: cert/onboard ready)
  - إذا !canSubmit → button disabled + tooltip عربي واضح يذكر السبب (Virtual/شهادة ناقصة/…)
  - إذا sandbox + integration_enabled=true → يسمح submit ويظهر label "إرسال Sandbox"
  - إذا production + integration_enabled=true → label "إرسال Production"

- ZatcaStatusBadge:
  - يدعم عرض حالة إضافية "Virtual" عندما integration_enabled=false في sandbox
  - ما تغيّرش الـ statuses الحالية، فقط أضف حالة عرض/Badge إضافية إن لزم

C) Edge Functions Guards (Critical)
- zatca-submit:
  - يستقبل mode + integration_enabled ضمن payload أو headers
  - إذا integration_enabled=false → RETURN 400 مع errorCode = 'ZATCA_VIRTUAL_MODE'
  - إذا mode='production' && cert/onboard not ready → RETURN 400 errorCode='ZATCA_NOT_READY'
  - إذا mode='sandbox' → يوجّه للـ sandbox base URL/config فقط
  - إذا mode='production' → يوجّه للـ production base URL/config فقط
  - ممنوع أي fallback صامت

- zatca-generate (Virtual support):
  - يجب أن يعمل حتى لو integration_enabled=false:
    يرجع data للتجربة (مثل XML/QR payload) بدون إرسال
  - لو integration_enabled=true: يسمح generate + sign بحسب التدفق الحالي

- zatca-sign / zatca-onboard:
  - لازم يحترم mode (sandbox/prod)
  - يمنع production operations بدون integration_enabled=true

D) Error Mapping (Arabic)
- في UI، اعرض رسائل عربية حسب errorCode:
  - ZATCA_VIRTUAL_MODE: "الوضع تجريبي (Virtual) — فعّل التكامل للإرسال"
  - ZATCA_NOT_READY: "إعدادات الشهادة/التسجيل غير مكتملة للإنتاج"
  - INVALID_MODE: "وضع ZATCA غير صالح"
  - ... حافظ على أي رسائل موجودة، فقط أضف الجديدة
