Implementation Prompt — Gold Purchase Excel Import Phase 1 Only

Context
تم حسم القرارات المعمارية والتصميمية التالية مسبقًا ويجب الالتزام بها بدون اجتهاد:

* لا يتم تعديل ImportPage الحالية الخاصة بالمجوهرات
* لا يتم reuse لمسار استيراد المجوهرات الحالي
* سيتم بناء GoldImportPage مستقلة بالكامل
* Gold purchase import path مستقل بالكامل عن Jewelry purchase import path
* Gold branches = branches where branch_type = 'gold' فقط
* Gold import uses:

  * WEIGHT_GROSS mandatory
  * STONE_WEIGHT optional
  * g_weight_net computed server-side
  * MAKING_CHARGE optional / nullable
  * gold item code format = GLD-{SEQ8}
* VAT on gold purchase import:

  * vat_rate configurable input
  * default = 0
  * لا يُعتبر هذا حسمًا نهائيًا للسياسة الضريبية، لكنه safe default للتنفيذ الآن

Scope
نفّذ فقط Gold Purchase Excel Import Phase 1.
لا تنفذ POS.
لا تنفذ sales.
لا تنفذ returns.
لا تنفذ pricing sessions.
لا تنفذ barcode printing.
لا تنفذ batch detail features beyond minimum import result.
لا توسّع خارج نطاق الاستيراد والشراء والمخزون والقيد الأساسي.

7. Gold Item Identity
   نفّذ generation مستقل لأكواد الذهب بصيغة:

* `GLD-{SEQ8}`

مثال:

Requirements:

* sequence مستقل عن المجوهرات
* exact uniqueness
* لا تضمّن karat أو month أو branch داخل الكود
* barcode field:

  * if provided in Excel use it
  * else generate from gold item code safely

8. Server-Side Weight Handling
   داخل gold import path:

* خزّن `g_weight_gross`
* خزّن `stone_weight` default = 0 إذا غائبة
* احسب `g_weight_net = g_weight_gross - stone_weight` في السيرفر فقط
* امنع أي net weight calculation في frontend
* ارفض أي صف لو net becomes <= 0
